# SSH 工具箱 — Git 下载与本地部署手册

本文说明如何从 GitHub 克隆 **SSH 工具箱**（`ssh-toolbox`），在 macOS 上完成安装、运行与可选打包。

- 仓库地址：https://github.com/langzf/ssh-toolbox  
- 适用系统：**macOS 12 及以上**（Apple 芯片 / Intel 均可，见下文架构说明）

### 更简单：npm 安装（推荐给他人的方式）

若已发布到 npm，对方只需：

```bash
npm install -g ssh-toolbox
ssh-toolbox
```

详见 [npm安装说明.md](./npm安装说明.md)。**对方电脑需已安装 Node.js 18+。**

---

## 一、环境准备

### 1.1 安装 Git

终端执行：

```bash
git --version
```

若未安装，可安装 Xcode 命令行工具：

```bash
xcode-select --install
```

或从 [git-scm.com](https://git-scm.com/download/mac) 安装 Git。

### 1.2 安装 Node.js（18 或更高）

```bash
node --version
npm --version
```

推荐使用 **Node.js 18 LTS 或 20 LTS**：

- 官网：https://nodejs.org/  
- 或使用 Homebrew：`brew install node`

### 1.3 本机 SSH（连接远程服务器时）

- 连接**远程 Linux 服务器**：确保对方已开启 SSH（端口一般为 22）。  
- 连接**本机 Mac**：在 **系统设置 → 通用 → 共享** 中开启 **远程登录**。

## 二、从 GitHub 下载代码

### 2.1 选择存放目录

例如用户主目录下的 `Projects`：

```bash
mkdir -p ~/Projects
cd ~/Projects
```

### 2.2 克隆仓库

**HTTPS（通用）：**

```bash
git clone https://github.com/langzf/ssh-toolbox.git
cd ssh-toolbox
```

**SSH（已配置 GitHub SSH 密钥时）：**

```bash
git clone git@github.com:langzf/ssh-toolbox.git
cd ssh-toolbox
```

克隆完成后，项目根目录应包含：`package.json`、`main/`、`src/`、`README.md` 等。

### 2.3 后续更新代码

```bash
cd ~/Projects/ssh-toolbox
git pull
npm install    # 若 package.json 有变更，建议执行
```

## 三、本地部署（开发运行方式）

此方式适合日常使用与尝鲜，改代码后重启即可生效。

### 3.1 安装依赖

在项目根目录执行：

```bash
npm install
```

首次安装会下载 Electron 等依赖，可能需要几分钟。

### 3.2 自检（可选）

```bash
npm run check
```

用于检查依赖与基本文件是否齐全。

### 3.3 启动应用

**方式 A — npm：**

```bash
npm start
```

**方式 B — 启动脚本：**

```bash
chmod +x start.sh
./start.sh
```

成功后会打开独立窗口，标题为 **SSH 工具箱**。

### 3.4 关闭应用

使用 **Cmd + Q** 退出，或关闭窗口后确认进程已结束。

## 四、可选：打包为 macOS 应用程序

若希望像普通 Mac 软件一样从 **启动台 / 应用程序** 打开，可本地打包。

### 4.1 打包

```bash
cd ~/Projects/ssh-toolbox
npm run pack
```

完成后生成：

```text
dist/mac-arm64/SSHToolbox.app
```

- 在 **Apple 芯片（M 系列）** Mac 上打包，一般为 `mac-arm64`。  
- 在 **Intel** Mac 上打包，目录可能为 `dist/mac/`。  
- 若需给 Intel 机器使用，请在对应架构的 Mac 上执行 `npm run pack`，或使用 electron-builder 指定架构（见 `package.json` 中 `build.mac.target`）。

### 4.2 安装到「应用程序」

1. 在访达中打开 `dist/mac-arm64/`（路径按实际为准）。  
2. 将 **`SSHToolbox.app`** 拖入 **应用程序** 文件夹。  
   - 访达默认可能是 **移动**（拖走后 `dist` 里会变空），属正常。  
   - 若要在 `dist` 中保留副本：拖入时按住 **Option（⌥）**。  
3. 或终端复制：

```bash
cp -R dist/mac-arm64/SSHToolbox.app /Applications/
```

### 4.3 首次打开（未签名应用）

macOS 可能提示无法验证开发者：

1. **右键** `SSHToolbox.app` → **打开** → 再点 **打开**；或  
2. **系统设置 → 隐私与安全性** → **仍要打开**。

若提示「无法打开」，可清除隔离属性后重试：

```bash
xattr -cr "/Applications/SSHToolbox.app"
```

### 4.4 更新已安装的 .app

```bash
cd ~/Projects/ssh-toolbox
git pull
npm install
npm run pack
```

退出正在运行的旧版 → 用新的 `SSHToolbox.app` **替换**「应用程序」中的旧文件 → 再打开。

## 五、基本使用说明

### 5.1 添加服务器

1. 侧栏 **库存 → 服务器**  
2. 点击 **⊞+** 新建  
3. 填写主机名、端口（默认 22）、用户名（默认 `root`）、密码或私钥  
4. **保存**（仅入库）或 **连接**（立即 SSH）

密码可勾选「记住密码」，加密保存在本机（`~/Library/Application Support/SSH 工具箱/`）。

### 5.2 终端会话

- 连接后可在侧栏 **会话** 中切换多个标签。  
- 主区域为完整 SSH 终端。

### 5.3 SFTP 文件传输

1. 先建立 SSH 连接。  
2. 工具栏点击 **SFTP**。  
3. **双击文件夹** 进入目录；选中文件后 **下载**；**上传** 选择本机文件。  
4. 支持新建文件夹、删除、刷新、返回上级目录。

### 5.4 设置

侧栏 **⚙ 设置** 可修改默认端口、终端字号、外观主题。

## 六、数据与配置存放位置

应用数据**不会**打进 `.app`，也**不会**随 `git clone` 下载，仅保存在本机用户目录，例如：

```text
~/Library/Application Support/SSH 工具箱/
```

其中可能包含：已保存服务器列表、加密后的密码、片段、设置等。

- **换电脑**：需重新添加服务器（或自行备份上述目录）。  
- **分享 .app 给他人**：对方得到的是「干净」应用，不含你的连接信息。

## 七、常见问题

| 现象 | 处理建议 |
|------|----------|
| `npm install` 很慢或失败 | 检查网络；可配置 npm 镜像；删除 `node_modules` 后重装 |
| `npm start` 无窗口 / 报错 `ELECTRON_RUN_AS_NODE` | 使用项目内 `start.sh`，或执行 `unset ELECTRON_RUN_AS_NODE` 后再 `npm start` |
| 连接被拒绝 `ECONNREFUSED` | 确认目标主机 SSH 已开启、端口正确、防火墙放行 |
| 认证失败 | 检查用户名、密码、私钥路径及私钥口令 |
| 打包后双击「无法打开」 | 使用 `SSHToolbox.app`（新版本）；右键 → 打开；执行 `xattr -cr` |
| SFTP 无法列出目录 | 确认服务器支持 SFTP 子系统（OpenSSH 默认通常支持） |
| `git pull` 冲突 | 先提交或备份本地修改，再合并远程更新 |

## 九、安全提示

- 不要将服务器密码、私钥、GitHub Token 提交到 Git 仓库。  
- 仓库已配置 `.gitignore` 排除 `node_modules`、`dist`、`.env` 等。  
- 仅在可信 Mac 上使用「记住密码」功能。
